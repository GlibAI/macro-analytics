üìä Database Overview

The Macro Analytics database stores financial transaction data for analysis and reporting. The main table transactions contains detailed information about all bank transactions, including amounts, dates, account information, and categorization.

üéØ Tips for Better Queries

1. Be Specific with Dates Use clear date formats like "January 15, 2025" or "2025-01-15". You can also use relative dates like "last 30 days".
2. Use Column Names in Queries While the AI can understand natural language, mentioning specific column names improves accuracy. For example: "Show transactions where category is 'Salary'" instead of "Show salary transactions".
3. Specify Aggregations If you want totals or counts, be clear: "Total amount by category" or "Count of transactions".
4. Know Your Data Values Common values:
  • Type: Credit, Debit
  • Account Type: Savings, Current, Operating
  • Mode: Check, NEFT, RTGS, Deposit
  • Categories: Salary, Utilities, Transfer, etc.
‚ö†Ô∏è Note About Account Numbers Account numbers are masked for security. You cannot search by full account numbers, but you can still query by account_name or other attributes.
